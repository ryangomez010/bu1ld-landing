# Agent Operating Notes

This workspace currently centers on The BU1LD Nexus and a broader research/product portfolio. Follow these rules before making changes:

1. Preserve user work. The BU1LD tree has active page, route, SEO, and public asset changes. Do not revert, overwrite, or normalize those changes unless explicitly asked.
2. Treat portfolio work as evidence-gated. Do not promote a project from idea/prototype to release status unless code, tests, artifacts, and reproducible commands support the claim.
3. Keep BU1LD and FinanceMeta distinct. BU1LD is a research-and-builder institution platform; FinanceMeta is a finance education/member platform. Shared mechanics are acceptable, but brand, information architecture, claims, and journeys must not collapse into a generic community template.
4. Supabase changes require schema, RLS, server-side authorization, denial states, and verification scripts to stay synchronized.
5. Research claims require raw artifacts, fair baselines, fixed seeds, and clear limitations. Synthetic smoke tests prove execution, not external validity.
6. Use `research/PROJECT_REGISTRY.yaml` as the canonical project map for this pass.
7. Use `docs/NEXT_HANDOFF.md` and `research/PERSISTENT_MEMORY.md` before continuing future implementation.
