1. Apply and verify production schema through phase32.
2. Configure auth URLs/providers and server secrets.
3. Run visitor/member/lead/reviewer/admin project lifecycle smoke.
4. Deploy to Cloudflare and verify headers, health, metadata, legal pages, auth, and core writes.

## Days 8–30: controlled validation

1. Operate 10 real projects with owners and current milestones.
2. Instrument 20 contributors through evidence submission.
3. Measure contribution review turnaround, overdue milestones, stalled projects, and first-to-second contribution return.
4. Add Playwright coverage from the observed production journeys.

## Days 31–90: operational proof

1. Archive abandoned projects and publish only evidence-backed outcomes.
2. Add a single operational stats RPC if admin count-query latency warrants it.
3. Publish reviewed contribution portfolios and reproduction reports.
4. Review retention and program demand before adding product scope.

No new divisions, dashboards, or venture features should be added until the release gate has external evidence.
